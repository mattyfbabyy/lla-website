AUTO-PUBLISHING: NO SETUP REQUIRED

The old version of this file described a Vercel Deploy Hook step. That is no
longer needed. The workflow now publishes on its own:

- Every morning at 7:30 AM Chicago, GitHub checks whether any article in the
  articles folder is dated today
- If yes, it makes a tiny "publish" commit, Vercel rebuilds automatically,
  and the post goes live
- If no, it does nothing

It works the moment the .github folder is in your repo. Nothing to configure.

To publish manually anytime (for example on launch morning if you want to
watch it happen): repo > Actions tab > "Publish scheduled posts" > Run workflow.

One check after your first push: repo > Settings > Actions > General >
Workflow permissions should be set to "Read and write permissions"
(this is the default on most repos; only change it if the Action fails).
